## Diagnosis Process

1. Read the error message — what failed?
2. Check the rendered spec — which line?
3. Determine root cause:
   - Method not found → check Method Registry for correct method name
   - Element not found → wrong page context, switch the "page" field
   - Multiple elements → use an index-based method (e.g. clickAddButtonByIndex)
   - Wrong assertion value → use exact text from App Model appMessages
   - Disabled button → missing prerequisite step (check App Model stepper enabledWhen)
   - State not seeded → add stateSeeding field from App Model
4. Apply MINIMAL fix

## Rules

- Output the COMPLETE spec data matching the schema (NOT page objects)
- Use ONLY methods from the Method Registry
- After navigation actions, switch "page" to the new page
- If a test needs pre-existing data (e.g. existing user), set the "stateSeeding" field
- If a test case testData has stateSeeding, ALWAYS pass it through
- Use EXACT assertion text from App Model appMessages
- Keep ALL tests — do NOT remove passing ones
